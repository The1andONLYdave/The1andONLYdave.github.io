    Bring dein Chromebook in den Entwicklermodus (Developer Mode).

    Achtung: Dies löscht alle lokalen Dateien, also vorher alles auf SD-Karte, USB oder Google Drive sichern. Du musst danach auch dein Googlekonto und dein Wlanpasswort neu eingeben, halte es also bereit.
    Achtung: dies macht dein Chromebook unsicher, denke später daran ein Passwort für das Konsolenlogin zu vergeben, damit nicht jeder direkt an die Daten aller Nutzer kommt ohne Passwortabfrage.

    Drücke dazu bei den meisten Chromebooks kurz auf den Ein/Ausschalter während du die Tasten "esc" und "aktualisieren" (F3, die mit dem Pfeil im Kreis) gedrückt hältst. *
    Im folgenden Bildschirm drückst du einfach die Taste "d" während du "strg" gedrückt hältst.
    Nach ca 8 Minuten startet dein Chromebook neu. Richte dann einen Account ein oder nutze den Gastmodus und gehe zu Schritt 2. Du musst nun bei jedem (Neu-)Start deines Chromebooks strg und "d" drücken oder ca eine Minute warten bis es hochfährt. Drückst du wie aufgefordert die Leertaste und Bestätigst mit Enter wird dein komplettes Chromebook zurückgesetzt. Das heisst du verlierst nicht nur wie hierbei alle Daten sondern auch das installierte und eingerichtete Linux ist komplett weg.

    Merke: es gibt später drei Passwörter: Dein normales Google Passwort, das Konsolenpasswort von ChromeOS und das Linux-Passwort.

    [*] funktioniert dies nicht, schau bitte in folgender Liste nach deinem Chromebook / deiner Chromebox nach wie du in den Developer Modus kommst: ChromeOS Wiki

    Weiter zu Schritt 3: Linux





    Um ein Kommandozeilenfenster zu öffnen kannst du nun im Google Chrome jederzeit "t" drücken während du "strg" und "alt" hältst.
    Oder du lädst dir folgende Erweiterung runter und bekommst ein Symbol dafür, welches du unten in der Leiste von Chrome OS anheften kannst: Crosh Erweiterung im Google Web Store. Damit diese Erweiterung ohne Probleme funktioniert benötigst du auch diese Erweiterung noch Secure Shell im Google Web Store

    Öffne nun auf deine Art ein Kommandozeilenfenster und tippe "shell" ein. Wenn dies dein erstes Mal in der Kommandozeile ist, alle Befehle werden immer mit der nEtertaste bestätigt und am besten immer Kopieren und Einfügen, da Stelle und Anzahl der Leerzeichen und Groß-/Kleinschreibung wichtig ist. Einfügen kannst du in der Chrome OS Kommandozeile einfach mit der mittleren Maustaste oder einem Klick mit zwei Fingern auf das Touchpad bei den meisten Chromebooks.

    Tippe nun "sudo chromeos-setdevpasswd" um ein sicheres Passwort für die Kommandozeile zu setzen. Das Passwort wird beim Tippen nicht angezeigt, auch nicht als ***.

    Nun starten wir die Linux Installation. Um ein aktuelles Ubuntu Linux zu bekommen mit grafischer Ausgabe, Unterstützung für die Chromebooktastatur, und einem schlanken aber vollständigen Desktop und der Möglichkeit Kopieren und Einfügen von Text zwischen Linux und ChromeOS benutze folgende Befehle (ohne ""):
    "cd"
    "cd Downloads"
    "sudo sh crouton -r xenial -t xorg,extension,xfce-desktop,keyboard
    Dies wird nun eine Weile dauern, bis alles heruntergeladen (ca 1GB) und installiert ist. Je nach Chromebook meistens zwischen 30 und 120 Minuten. Am Ende wirst du aufgefordert ein Nutzername und ein Passwort für das Linux festzulegen.
    Danach starten wir das Linux erstmal ohne die grafische Oberfläche und installieren noch ein paar nützliche Programme und deutsche Sprachpakete mit: "sudo enter-chroot" und den folgenden Befehlen:

    Nützliche Programme und die Sprachpakete
    sudo apt-get install git dialog vlc pcmanfm gimp-help-de gimp nano language-pack-gnome-de language-pack-gnome-de-base language-pack-de-base language-pack-de openjdk-8-jre icedtea-8-plugin

    (optional)Emulatoren und Spiele:
    sudo add-apt-repository ppa:libretro/stable
    sudo apt-get update
    sudo apt-get install dosbox wesnoth-1.12 steam retroarch libretro-genesisplusgx libretro-mednafen-psx libretro-quicknes libretro-nestopia libretro-mupen64plus libretro-snes9x-next libretro-vba-next libretro-vbam libretro-gpsp

    (optional)Flashplayer:
    sudo add-apt-repository "deb http://archive.canonical.com/ubuntu $(lsb_release -sc) partner"
    sudo apt-get update
    sudo apt-get install adobe-flashplugin

    (optional)Windowsemulator für einzelne Spiele:
    sudo apt-get install wine playonlinux winetricks

    (optional)Meine Lieblinge
    sudo apt-get install conky-all atop htop terminator mc gparted

    (optional)Teamviewer:
    Lade dir Teamviewer für Ubuntu herunter
    sudo dpkg -i ~/Downloads/teamviewer_i386.deb
    dann ein "sudo apt-get install -f"
    dann nochmals "sudo dpkg -i ~/Downloads/teamviewer_i386.deb"

    (optional)Ein toller Texteditor:
    Download von Sublime Text Version 3 Ubuntu amd64
    sudo dpkg -i ~/Downloads/sublime-text_build-3126_amd64.deb

    Danach noch das System auf Deutsch umstellen:
    sudo update-locale LANG="de_DE.UTF-8" LANGUAGE="de:en"
    und mit "exit" das Linux verlassen.

    Nun kannst du das Linux jederzeit in einer Kommandozeile (das mit strg, alt und t) starten mit folgenden zwei Befehlen:
    shell
    sudo startxfce4

    Wechseln zwischen Linux und Chrome OS kannst du, in dem du die Tasten "strg", "alt", "shift"(Großschreiben) gedrückt hältst und die Pfeile (F1 und F2 Tasten) drückst. Pfeil nach Links wechselt zu ChromeOS, Pfeil nach Rechts zu Linux.
    Sobald du mit Linux fertig bist, oder dein Chromebook neustarten willst, wähle in Linux bitte die Option "Herunterfahren" - Keine Angst, dies sorgt nur dafür das Linux beendet wird ohne Datenverlust und du zurück zu ChromeOS gelangst - ChromeOS wird hierbei nicht mitheruntergefahren.

    Noch ein Tipp für mehr Geschwindigkeit der Linuxoberfläche: Öffne im Linux die Anwendung "Feineinstellungen der Fensterverwaltung" und wähle auf dem letzten Tab "Komposit" die oberste Option "Anzeigenkomposit aktivieren" ab, so dass alles auf dieser Seite deaktiviert ist.

    Willst du dein Linux einmal komplett sichern, beende es bitte und nutze in der Kommandozeile folgende Befehle:
    shell
    sudo edit-chroot -b xenial
    Eine Wiederherstellung kannst du dann z.Bsp. mit "sudo sh -e crouton -r xenial -t xorg,extension,xfce-desktop,keyboard -f ~/Downloads/xenial-20170206-1234.tar.gz" machen.

    Häufige Probleme:

    Probleme mit schwarzem Bildschirm beim Linuxstart (nur wenn erforderlich, u.a. Acer R11 - Modelle):
    "sudo cp /mnt/stateful_partition/crouton/chroots/xenial/etc/crouton/xorg-intel-sna.conf /mnt/stateful_partition/crouton/chroots/xenial/usr/share/X11/xorg.conf.d/20-crouton.conf" und dann in dieser datei noch folgende zeile ändern "Option "AccelMethod" "uxa"" (uxa statt sna)
